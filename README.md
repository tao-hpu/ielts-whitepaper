# 雅思白皮书

> 把雅思讲成一套**可以被逆向工程的评分系统**。
> 说人话：**从「什么是雅思」到「怎么练」的系统化听说读写攻略。** 📄✎

**🔗 在线访问 → [ielts.fim.ai](https://ielts.fim.ai)**

这不是又一份机经堆砌，也不是"背了就能过"的话术。全书只围绕一个念头展开：**分数不是玄学，是考官照着评分标准（band descriptors）逐条勾出来的**。看懂这把尺，你就能把模糊的"提高英语"翻译成清晰的"满足哪几条要求"——然后按图施工。

**读完你能**：知道雅思到底怎么打分、6.5 该怎么配分、听说读写每一项练什么、以及一套"诊断 → 处方 → 复训"的备考闭环。风格对标 [linalg → attention](https://l2a.fim.ai)，只不过这次搭的是应试方法学。

## 一条主线

> 雅思不是一场英语考试，是一套可以被逆向工程的评分系统。

从「认识考试」→「分数的逆向工程」→「听读：确定性的分」→「说写：守住的分」→「怎么练、怎么上考场」，一路把评分标准倒推成可执行的训练。

## 两个核心念头

- **分数 = 可勾选标准的总和**：从 band descriptors 倒推每一项该练什么。
- **听说读写共享一条底层链**：输入端（听/读）识别的同义替换，就是输出端（说/写）的高分表达。

## 大纲（5 部分 · 16 节）

1. **认识这场考试** — 什么是雅思 · 九分制这把尺 · 考官怎么打分
2. **分数的逆向工程** — 分数是凑出来的（田忌赛马） · 焊住四项的同义替换链
3. **听与读：确定性的分** — 听力 · 阅读题型地图 · T/F/NG 的分界线 · Matching（规划中）
4. **说与写：守住的分** — 口语框架 · 写作两个 Task · Task 1 语言库 / Task 2 结构（规划中）
5. **怎么练，怎么上考场** — 诊断→处方→复训 · 十周计划 · 考场策略与材料

> ✎ 标记「规划中」的节点是后续要写的模块。真题只认官方《剑桥雅思》，本项目不分发任何真题内容。

## 技术栈

Vite + React 18 + TypeScript，多页靠 react-router，构建后 prerender 打成静态站（每节烤正确的 meta / og）。学院绿（Cambridge green）纸感排版，零重型依赖。

## 本地运行

```bash
pnpm install
pnpm dev      # http://localhost:5192
pnpm build    # 产物在 dist/
```

## 自托管

构建产物是一个纯静态站，`dist/` 丢给任何静态服务器都能跑。仓库自带 Docker 配置：

```bash
docker compose up -d --build   # 构建镜像并起容器，默认监听 127.0.0.1:5192
```

容器只监听本地端口，前面挂一层反向代理（nginx / Caddy 等）指到该端口即可对外。不想用 Docker，直接 `pnpm build` 后把 `dist/` 交给你惯用的静态托管也行。
